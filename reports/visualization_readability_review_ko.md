# AEROPATENT 전체 기획/시각화/가독성 리뷰

작성일: 2026-06-26

## 1. 총평

기획 방향은 좋다. 특히 특허분석에서 흔히 생기는 중복계상, 기간 해석, 공개국가/출원인국가 혼동, 법적 상태 과대해석을 분리한 점이 강하다. 현재 구조는 단순 데모가 아니라 연구기획/사업개발용 인텔리전스 대시보드로 발전시킬 수 있다.

다만 화면은 차트를 많이 넣는 것보다 "처음 30초 안에 무엇을 봐야 하는가"를 고정해야 한다. 첫 화면은 검색창 중심이 아니라 결론 중심이어야 하고, 그래프뷰는 예쁜 배경이 아니라 분석결과로 들어가는 탐색 장치여야 한다.

현재 기획 점수:

- 데이터 신뢰성 구조: 8.5 / 10
- 분석 목적 적합성: 8 / 10
- 시각화 방향성: 7.5 / 10
- 가독성: 7 / 10
- 구현 가능성: 8 / 10

가독성이 아직 7점인 이유는, 지표와 차트가 많아질수록 사용자가 "그래서 어디를 봐야 하지?"라고 느낄 수 있기 때문이다. 따라서 화면별 1차 질문을 고정해야 한다.

## 2. 기획에서 잘 된 점

1. 기간 기준이 현실적이다.

- 최근 10년: 장기 판세
- 최근 5년: 기본 대시보드
- 최근 3년: 급상승
- 최근 1년/1개월: 신규 공개 모니터링

특허는 공개 지연이 있으므로 1개월/1년만으로 동향을 판단하지 않는 것이 맞다.

2. 패밀리 기준을 메인으로 둔 것이 좋다.

같은 발명이 여러 국가에 공개되면 공개문헌 수가 부풀려진다. 기술 규모는 `family_id` 기준으로 보고, 국가 활동은 공개문헌 수로 보조하는 구조가 맞다.

3. 공개국가와 출원인국가를 분리한 것이 좋다.

`country_code`는 공개국가 기준이다. 경쟁국/위협 분석에는 `assignee_country_codes`가 보조로 필요하다. 이 둘을 분리한 것은 신뢰성 측면에서 중요하다.

4. 한국 공백 분석을 "후보"로 표현하는 방향이 좋다.

한국 취약 분야라고 단정하면 과하다. "세계 성장 대비 한국 신호가 약한 기회 후보"라고 표현해야 연구기획용으로 안전하다.

5. 월 0원 운영을 위해 BigQuery live dashboard를 피한 점이 좋다.

사이트는 BigQuery를 직접 때리지 않고, 월 1회 생성된 JSON/JSONL을 읽어야 한다.

## 3. 남은 위험

1. CPC-first 방식은 비용에는 좋지만 오탐이 있다.

넓은 CPC prefix는 관련 없는 문헌도 끌고 올 수 있다. 분야별 샘플 50건 검토가 필수다.

2. 상용항공 중심이라고 해도 방산/dual-use가 섞인다.

항공우주는 민간/방산 경계가 흐리다. 방산을 제외하지 않되, 화면에는 `dual-use` 또는 `defense_relevance` 태그를 붙여 해석을 돕는 것이 좋다.

3. 법적 상태 분석은 아직 약하다.

Google Patents Public Dataset만으로 active/enforceable 여부를 단정하면 안 된다. `grant_date`는 "권리화 신호"까지만 표현한다.

4. 그래프뷰가 과하면 가독성이 떨어진다.

초기부터 모든 특허 노드를 보여주면 은하수가 아니라 먼지처럼 보인다. 초기 graph는 분야, 국가, 상위 출원인만 보여주고, 특허 노드는 줌/클릭 후 노출한다.

## 4. 화면 구성 원칙

### 4.1 첫 화면의 질문

첫 화면은 이 질문 5개에 답해야 한다.

1. 전체 항공우주 특허 활동은 증가 중인가?
2. 어떤 분야가 가장 빠르게 커지는가?
3. 어느 공개국가가 어느 분야에서 강한가?
4. 한국 기회 후보는 어디인가?
5. 이번 달 새로 볼 만한 문헌은 무엇인가?

따라서 첫 화면의 구조는 아래 순서가 좋다.

```text
Top Filter Bar
KPI Strip
Field x Country Heatmap
10-year Field Trend
Country Portfolio
Korea Gap / Opportunity
Monthly Watchlist
```

검색은 첫 화면 주인공이 아니다. 검색은 분석을 보고 더 깊게 들어가고 싶을 때 쓰는 2차 기능이다.

### 4.2 KPI Strip

KPI는 5개만 둔다.

| KPI | 의미 |
|---|---|
| 전체 패밀리 | 기술 규모 |
| 최근 5년 성장률 | 현재 동향 |
| 급상승 분야 | emerging trend |
| 선도 공개국가 | 공개국가 기준 활동 |
| 한국 기회 후보 | 연구기획 액션 |

KPI는 카드처럼 과하게 꾸미지 말고, 얇은 구분선과 숫자 중심으로 보여준다. 각 KPI에는 반드시 기준을 붙인다.

예:

```text
최근 5년 성장률
priority_date 기준, family count
```

## 5. 차트별 구체화

### 5.1 분야 x 국가 히트맵

목적:

어느 분야가 어느 공개국가에서 강한지 한눈에 보여준다.

데이터:

- x축: `publication_country_code`
- y축: `field_id`
- 값: `family_count`
- 보조: `publication_count`, `confidence`, `recent_5y_growth`

가독성 규칙:

- rainbow 색상 금지
- 한 가지 sequential color scale 사용
- 셀 안 숫자는 family count만 표시
- tooltip에 공개문헌 수와 기준 설명 표시
- US, EP, JP, CN, KR 순서 고정

### 5.2 최근 10년 분야별 추세

목적:

장기 판세와 최근 변화가 같이 보이게 한다.

기본 차트:

- stacked area chart

보조 전환:

- line chart

가독성 규칙:

- 너무 많은 분야가 겹치면 상위 6개 + Other
- 범례는 오른쪽 끝 직접 라벨
- 기본 선택은 최근 5년, "10년 보기" 버튼 제공

### 5.3 국가별 분야 Mix Bar

목적:

각 공개국가가 어떤 분야 포트폴리오를 갖는지 보여준다.

차트:

- 100% stacked bar

가독성 규칙:

- 비중은 bar에서, 원수치는 tooltip에서
- 항상 "공개국가 기준" 배지 표시
- 출원인국가 기준으로 전환하면 색상은 유지하고 제목만 명확히 변경

### 5.4 한국 공백/기회 테이블

목적:

연구기획과 사업개발 액션을 만든다.

컬럼:

- 순위
- 분야
- 글로벌 최근 3년 모멘텀
- 한국 공개국가 비중
- 한국 출원인국가 비중
- 기회 점수
- 신뢰도
- 한 줄 해석

가독성 규칙:

- "한국 취약"이라고 쓰지 않는다.
- "한국 공백 후보", "한국 기회 후보"라고 쓴다.
- 숫자만 나열하지 말고 한 줄 해석을 붙인다.

예:

```text
위성통신/LEO 네트워크
글로벌 모멘텀은 높지만 KR 공개/출원인 신호가 낮아, 연구기획 후보로 우선 검토할 만함.
```

### 5.5 상위 출원인 랭킹

목적:

연구협력/사업개발/경쟁사 모니터링 후보를 찾는다.

차트:

- ranked horizontal bar

탭:

- 전체
- 민간기업
- 공공연구기관
- 한국 관련

가독성 규칙:

- 출원인명 alias normalization 필요
- family count 기본
- publication count 보조
- 상위 10개까지만 첫 화면에 표시

### 5.6 월간 Watchlist

목적:

최신 공개 문헌을 빠르게 훑는다.

표시:

- 제목
- 공개번호
- 공개국가
- 분야
- 출원인
- 왜 봐야 하는지

가독성 규칙:

- 첫 화면에는 8개 이하
- "publication_date 기준" 표시
- 중요한 문헌이 없으면 억지로 채우지 않는다

## 6. Graph View 구체화

Graph View는 전체 데이터의 예쁜 그림이 아니라, 분석에서 상세 보고서로 이동하는 탐색 인터페이스다.

### 6.1 초기 상태

초기에는 모든 노드를 보여주지 않는다.

표시 노드:

- 9개 field node
- 5개 country node
- 상위 20개 assignee node

숨김 노드:

- publication node
- 세부 CPC node
- 낮은 score family node

이유:

처음부터 모든 특허를 보여주면 그래프가 복잡해져 사용자가 의미를 읽지 못한다.

### 6.2 노드 크기

| 노드 | 크기 기준 |
|---|---|
| Field | family count |
| Country | 해당 기준 family count |
| Assignee | family count |
| Family | opportunity/impact score |
| Publication | 기본 작게, 선택 시 확대 |
| CPC | 연결 수 |

### 6.3 색상

색상은 field 기준으로만 강하게 쓴다.

- Field: 고정 accent color
- Country: 중립색 + 선택 시 강조
- Assignee: 조직 유형별 미세 구분
- Publication: 낮은 opacity
- 선택 노드: 흰색 또는 강한 outline

색을 너무 많이 쓰면 의미가 사라진다. 그래프에서는 크기, opacity, 거리, label 노출 단계로 정보를 나눠야 한다.

### 6.4 클릭 모션

1. 분야 노드 클릭
2. 카메라가 해당 field cluster로 이동
3. 관련 없는 노드는 opacity 15%로 감소
4. 관련 country/assignee/family 노드만 label 표시
5. 오른쪽 report drawer 열림

Drawer 내용:

- 분야명
- 최근 5년 성장률
- 선도 공개국가
- 한국 공백 후보 여부
- 상위 출원인
- 대표 패밀리 5개
- "상세 분석 보기" 버튼

### 6.5 그래프 모드

그래프에는 모드가 있어야 한다.

| 모드 | 목적 |
|---|---|
| 분야 지도 | field 중심 |
| 국가 비교 | country 중심 |
| 출원인 네트워크 | assignee 중심 |
| 한국 기회 | korea gap score 중심 |
| 월간 신규 | publication_date 기준 신규 노드 |

초기 기본값은 `분야 지도`가 가장 좋다.

## 7. 가독성 리뷰

현재 기획은 분석 내용은 좋지만, 화면에서 너무 많은 지표를 동시에 보여주면 읽기 어렵다. 따라서 아래 규칙을 적용해야 한다.

### 7.1 한 화면 한 질문

각 섹션은 질문 하나에만 답한다.

나쁜 예:

```text
국가별/분야별/출원인별/기간별/한국공백을 한 카드 안에 다 넣기
```

좋은 예:

```text
분야 x 국가 히트맵: 어느 나라가 어느 분야에서 강한가?
한국 공백 테이블: 한국 기회 후보는 어디인가?
```

### 7.2 숫자 기준 표시

모든 숫자 옆에는 기준이 필요하다.

- family count
- publication count
- priority_date 기준
- publication_date 기준
- 공개국가 기준
- 출원인국가 기준

### 7.3 텍스트는 설명이 아니라 해석

차트 위 설명문은 길면 안 된다.

좋은 문장:

```text
최근 5년 기준, 위성통신과 상용항공 전기추진이 가장 빠르게 증가했습니다.
```

나쁜 문장:

```text
본 그래프는 각 분야별 특허 패밀리 수를 국가별로 집계하여 색상으로 표현한 시각화입니다.
```

### 7.4 카드는 줄인다

분석 대시보드는 카드 모자이크가 되면 산만하다.

권장:

- KPI strip은 얇게
- 차트는 full-width section
- report card와 watchlist만 카드 사용
- Graph View는 full canvas

### 7.5 신뢰도 배지

분류 검증 전에는 지표에 `탐색` 배지를 붙인다.

배지:

- 높음: precision 0.9 이상
- 보통: precision 0.8 이상
- 탐색: 아직 샘플 검토 전

## 9. 구현 전 체크리스트

- `LIMIT` 없는 production 데이터로 지표 생성
- 분야별 샘플 50건 precision 검토
- 공개국가/출원인국가 toggle 라벨 확인
- 모든 차트에 date basis 표시
- 모든 차트에 family/publication basis 표시
- 월간 watchlist는 publication_date 기준 표시
- grant_date는 권리화 신호로만 표현
- Graph View 초기 노드 수 제한
- 모바일에서는 Graph View보다 보고서 카드 우선 표시

## 10. 결론

기획은 방향이 좋고, 신뢰성 보강도 잘 들어갔다. 이제 중요한 것은 차트를 많이 넣는 것이 아니라 사용자가 빠르게 결론을 읽게 하는 것이다.

가장 좋은 첫 경험은 다음 한 문장으로 요약된다.

```text
최근 5년 항공우주 특허는 어떤 분야가 커지고, 어느 공개국가가 강하며, 한국은 어디를 기회 후보로 봐야 하는가?
```

이 질문에 첫 화면이 답하면 가독성은 충분히 좋아진다.
